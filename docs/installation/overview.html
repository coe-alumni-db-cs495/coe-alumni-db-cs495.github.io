Installation Guide: Overview & Prerequisites

About This Application

The COE Alumni Database is an ASP.NET Core 8 application with a React TypeScript frontend. This installation guide will walk you through deploying the application on your own Ubuntu server.

Technology Stack

Component Technology Version
Backend ASP.NET Core 8.0
Frontend React with TypeScript Latest
Runtime Node.js 20.x
Database SQLite 3.x
Deployment GitHub Actions Self-hosted runner

Prerequisites

Before you begin, make sure you have:

Required:
  • An Ubuntu server (18.04 or later recommended) with sudo access
  • Basic command line knowledge
  • A GitHub account with access to the repository
  • At least 2GB of RAM and 10GB of disk space
Optional:
  • A domain name pointed to your server (for production deployments)
  • SSL/TLS certificate (for HTTPS access)

What You'll Install

This installation guide will help you set up the following components:

  1. System Dependencies: .NET 8 SDK, Node.js 20, SQLite 3, Git
  2. Service Account: A dedicated user (coe-alumni) to run the application
  3. GitHub Self-Hosted Runner: For automated deployments
  4. Systemd Service: To manage the application lifecycle
  5. Application Files: Deployed to /var/www/coe-alumniDB/
  6. Database: SQLite database at /var/www/coe-alumniDB/DB/AlumniDatabase.db
  7. Optional: Nginx: Reverse proxy for domain-based access

Installation Time

Expect the complete installation to take approximately 30-45 minutes, depending on your server's internet connection and your familiarity with Linux administration.

Installation Steps

The installation process is broken down into the following sections:

  1. Server Setup - Create service account and install dependencies
  2. GitHub Runner Setup - Configure automated deployment
  3. Application Deployment - Deploy and start the application
  4. Configuration - Optional database and web server setup
  5. Operations & Maintenance - Learn how to manage the application
  6. Troubleshooting - Common issues and solutions
  7. Quick Reference - Command cheat sheet

Security Considerations

Important Security Notes:
  • The application runs under a dedicated service account with limited privileges
  • Database file permissions are set to 600 (owner read/write only)
  • The application listens on localhost:5000 by default (not exposed to the internet)
  • Consider setting up a firewall (ufw) and configuring Nginx with SSL for production use
